35/*
36To be able to define callbacks of different data types without
37having to define a million different individual callbacks, a
38more abstract structure is needed.
39
40For this, the GUICallbackDatum struct is introduced. The callback
41code can then cast the void pointer pData back to its original
42type which can be inferred from the Type attribute in form of the enum
43GUIInputType (and also, the callback code should already
44know which data types it expects).
45
46The GUICallbackObject is what will actually be passed to the
47callback function. It includes the FormID of the set of options
48it is registered to in order to allow for the same object to
49potentially receive multiple callbacks. FormIDs are set during
50initialization/creation of the GUI. Likewise, the integer indices
51of the Data map are the OptionIDs registered during set up of
52the respective form.
53
54The callbacks themselves are handled through CForge's ITListener
55and ITCaller interface templates, with the GUICallbackObject data
56type.
57 */
